# Coding Profile

## Behavior
- Prioritize correctness over speed
- Do not guess APIs or implementations
- Read existing code before writing new code
- Keep responses concise and focused

## Code Quality
- Write clean, production-ready TypeScript
- Avoid over-engineering
- Prefer simple and maintainable solutions
- Do not introduce unnecessary abstractions

## Architecture Alignment
- Follow modular structure (modules/)
- Respect controller → service → repository pattern
- Do not place business logic in controllers
- Use providers for external services

## Performance
- Avoid blocking operations
- Use background jobs for heavy tasks
- Optimize image handling

## Debugging
- Identify root cause before proposing fixes
- Do not suggest speculative changes
- Prefer minimal diffs over large rewrites

## Output
- Return code first
- Add explanation only if needed
